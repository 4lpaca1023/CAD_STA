# 鎖存器時序分析報告

## 1. 時序參數摘要
根據 PrimeTime 分析報告（`sky.rpt` 和 `paths.rpt`），鎖存器（`sky130_fd_sc_hd__dlxtn_1`）的關鍵時序參數總結如下：

| 參數 | 符號 | 值 (ns) | 描述 |
| :--- | :--- | :--- | :--- |
| **D-to-Q prop** | **$t_{pdq}$** | **25.92** | 從數據變化到 Q 變化（上升）的最大延遲。 |
| **D-to-Q cont.** | **$t_{cdq}$** | **13.64** | 從數據變化到 Q 變化（下降）的最小延遲。 |
| **建立時間** | **$t_{setup}$** | **-0.17** | 數據在時鐘邊緣之前必須穩定的最小時間。 |
| **保持時間** | **$t_{hold}$** | **-0.13** | 數據在時鐘邊緣之後必須穩定的最小時間。 |
| **時鐘到 Q prop** | **$t_{pcq}$** | *N/A* | 該路徑未觸發（鎖存器處於透明狀態）。 |

---

## 2. 數據來源與證據

### A. $t_{pdq}$（prop）
*   **來源：** `latch/sky.rpt` - `report_delay_calculation -max`
*   **值：** **25.9169 ns**（上升延遲）
*   **分析：** 巨大的延遲是由於過大的輸出負載（4.00 pF）相對於需求（0.16 pF），導致外插。
```text
從引腳：l1/D
到引腳： l1/Q
...
 上升延遲
  單元延遲 = 25.9169（以庫單位計）
    表格索引：
    (X) 輸入引腳過渡時間 = 0.2
    (Y) 輸出網絡總電容 = 4.00033
...
上升延遲 = 25.9169
```

### B. $t_{cdq}$（cont.）
*   **來源：** `latch/sky.rpt` - `report_delay_calculation -min`
*   **值：** **13.6407 ns**（下降延遲）
```text
 下降延遲
  單元延遲 = 13.6407（以庫單位計）
    表格索引：
    (X) 輸入引腳過渡時間 = 0.25
    (Y) 輸出網絡總電容 = 4.00033
...
下降延遲 = 13.6407
```

### C. $t_{setup}$（建立時間）
*   **來源：** `latch/paths.rpt` - 最大路徑（建立檢查）
*   **值：** **-0.17 ns**
*   **說明：** 負的建立時間表示數據可以在鎖存邊緣稍後到達。
```text
  時間借用信息
  ------------------------------------------------------------------
  tau2015_clk 標稱脈衝寬度                        25.00 
  時鐘延遲差異                                    0.00 
  庫建立時間                                    -0.17 
  ------------------------------------------------------------------
```

### D. $t_{hold}$（保持時間）
*   **來源：** `latch/paths.rpt` - 最小路徑（保持檢查）
*   **值：** **-0.13 ns**
```text
  時鐘重合悲觀值                                   0.00      50.00
  庫保持時間                                    -0.13      49.87
  數據需求時間                                              49.87
```
